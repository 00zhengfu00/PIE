源码编译 PIE

本文档介绍如何从最新的源代码构建PIE。 如果您想获得尚未将其发布到发行版中的功能或bug修复，这对你来说很有用。 如果您想自己修改代码来尝试修复bug或实现新功能，那么这里也是你必须读的。 如果按照我说的做了，请考虑将您的更改提交到主项目，以便我们都可以从中受益！

准备条件

建立PIE是直截了当的。您需要安装以下工具：:

  1. Git, 如果您要从Git存储库中检出源代码
  2. Java 运行时, 版本 >=6
  3. Ant, 用于执行构建

安装这些准备条件后，请执行以下步骤：

获取源码

你有两种方式获取源码:

  1. 使用Git检出源码: 
    git clone git://github.com/lojjic/PIE.git
cd PIE
    如果你想要2.x beta分支（master），那么你完成了; 如果你想要切换1.x稳定分支，然后运行： 
    git checkout 1.x
    or:
  2. 下载源的存档:
    http://github.com/lojjic/PIE/archives/master (2.x beta 分支)
    http://github.com/lojjic/PIE/archives/1.x (1.x 稳定版 分支)

执行构建

从的本地副本的源代码根目录中，在终端中调用以下命令:

ant

T这将执行构建，创建一个名为“build/”的新目录。 此目录将包含以下文件内容 ：

  • PIE.htc
  • PIE_IE678.js
  • PIE_IE9.js
  • PIE_uncompressed.htc
  • PIE_IE678_uncompressed.js
  • PIE_IE9_uncompressed.js

要使用这些文件，将它们复制到服务器上的目录中，并引用PIE.htc（或 PIE_uncompressed.htc进行调试）。 或者，如果你想使用 PIE JS版本，那么您可以使用脚本标签包含PIE_IEx.js文件（请参阅 PIE JS 文档查询详情.)
